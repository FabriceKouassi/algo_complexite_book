🧠 Synthèse

Table de hachage : hash(clé) → index dans tableau. Collision : deux clés différentes → même index. Chaining : liste à chaque index. Open addressing : trouver un autre index libre. O(1) en moyenne, O(n) au pire. Facteur de charge : ratio taille/capacité.

💼 Application
🔑 Dans mon système de cache, je devais stocker des sessions utilisateur (clé: ID, valeur: données). Avec une table de hachage, les recherches sont quasi instantanées. J'ai aussi dû gérer l'expiration (éviction LRU) et la rehachage quand la table se remplit : ces détails m'ont forcé à comprendre les vraies implications du hachage.
