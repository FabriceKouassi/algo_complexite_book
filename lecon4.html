« Cette semaine, j'ai exploré listes chaînées, piles, files, arbres binaires. Ce qui m'a marqué : l'impact sur la complexité selon l'opération (accès O(1) dans un tableau vs O(n) dans une liste). 🧠 Synthèse

Tableau : accès O(1), insertion O(n). Liste chaînée : insertion/suppression O(1) si on a le pointeur, recherche O(n). Arbre binaire de recherche : O(log n) en moyenne si équilibré. Le bon choix dépend des opérations fréquentes.

💼 Application
🗂️ Je développe un correcteur orthographique. Pour le dictionnaire, j'ai d'abord utilisé une liste (recherche lente). Après ce cours, j'ai implémenté un arbre binaire (puis un arbre équilibré) : la recherche de mots est devenue très rapide. J'ai aussi utilisé une pile pour la fonction annuler.
🔍 Auto-évaluation

✅ maîtrisé

Choisir entre tableau, liste, pile/file selon le besoin.

⚠️ à améliorer

Implémentation d'arbres équilibrés (AVL) et analyse dans les pires cas.

📈 stratégie

Coder un arbre AVL et comparer les performances avec un arbre binaire simple.
